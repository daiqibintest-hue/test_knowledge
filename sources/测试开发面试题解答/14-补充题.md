## Python 基础

**1. 深拷贝 vs 浅拷贝**：浅拷贝（`copy.copy`）只复制顶层，嵌套对象仍共享引用；深拷贝（`copy.deepcopy`）递归复制所有层级，完全独立。测试中构造多组相似数据时要用深拷贝，否则改一个影响全部。

**2. list / tuple / set / dict**：list 有序可变；tuple 有序不可变（可做 dict key）；set 无序、元素唯一、用于去重；dict 键值对、键唯一、查找 O(1)。

**3. 装饰器**：在不改原函数代码的前提下增强功能的语法糖。测试框架里大量用：`@pytest.fixture`、`@pytest.mark.parametrize`、`@allure.step`，以及自定义重试/计时装饰器。

**4. 生成器 vs 迭代器**：迭代器是实现了 `__iter__`/`__next__` 的对象；生成器是用 `yield` 简化创建迭代器的方式，**惰性求值、省内存**，适合处理大数据集。

**5. 多线程 vs 多进程**：多线程共享内存、适合 **IO 密集**（接口请求、文件读写）；多进程各自独立内存、能利用多核、适合 **CPU 密集**。接口自动化并发用多线程即可。

**6. GIL**：全局解释器锁，同一时刻 CPython 只允许一个线程执行字节码，导致多线程不能真正并行计算。所以 CPU 密集用多进程，IO 密集多线程仍有效（IO 等待时会释放 GIL）。

**7. 异常捕获**：`try/except/else/finally`，`except` 按异常类型分别处理，`finally` 做资源释放（关连接、关文件）。要捕获具体异常而非裸 `except`。

**8. `*args` vs `**kwargs`**：`*args` 收集位置参数成 tuple；`**kwargs` 收集关键字参数成 dict。请求封装里 `**kwargs` 透传 params/json/headers 非常常用。

**9. 类方法 / 静态方法 / 实例方法**：实例方法第一个参数 `self`（访问实例）；类方法 `@classmethod` 第一个参数 `cls`（访问类）；静态方法 `@staticmethod` 不依赖类或实例，就是放在类里的普通函数。

**10. 上下文管理器**：实现 `__enter__`/`__exit__`（或用 `@contextmanager`）的对象，配合 `with` 自动管理资源（`with open(...)`、数据库连接），保证异常时也能释放。

## HTTP 与接口基础

**1. GET vs POST**：GET 查询、参数在 URL、可缓存；POST 提交数据、参数在 body、不缓存。语义上 GET 幂等、POST 非幂等，但 **是否幂等取决于服务端实现**——POST 也可以通过幂等号/唯一请求 ID 做成幂等。

**2. PUT vs PATCH**：PUT 全量更新（替换整个资源、幂等）；PATCH 部分更新（只改部分字段）。

**3. 状态码**：401 未认证 / 403 已认证无权限 / 404 资源不存在 / 429 请求过多被限流 / 500 服务器内部错误 / 502 网关收到无效响应 / 503 服务不可用（过载或维护）。

**4. Cookie / Session / Token**：Cookie 存客户端；Session 存服务端、靠 Cookie 里的 sessionId 关联；Token 适合分布式和前后端分离——其中 **JWT 是自包含的**（用户信息编码在 token 里、服务端无需存储即可校验），而 **opaque token（不透明令牌）本身无意义，仍依赖服务端存储/查询** 来校验有效性。

**5. HTTPS 原理**：HTTP + TLS，通过 **非对称加密协商对称密钥**，再用对称密钥加密传输，配合证书校验服务端身份，保证机密性和完整性。

**6. 接口幂等性**：同一请求执行一次和多次结果一致。下单、支付要做幂等（防重复扣款），常用唯一请求号/token 去重。

**7. 如何测接口重复提交**：用同一幂等号/请求并发或连续提交多次，断言只生成一条记录、不重复扣减。

**8. 如何测接口限流**：短时间高频请求，断言超过阈值后返回 429/限流响应，且恢复后能正常访问。

**9. 如何测签名和时间戳**：篡改签名应被拒；过期时间戳应被拒；重放同一请求应被拒（防重放）。

**10. 如何测文件上传/下载**：上传测大小限制、格式校验、超大文件、并发；下载测内容完整性（MD5 比对）、断点续传、权限控制。

## SQL 与数据库

**1. inner / left / right join**：inner 只返回两表匹配的行；left 返回左表全部 + 右表匹配（右无则 NULL）；right 反之。

**2. group by vs having**：group by 分组；having 对 **分组后** 的结果过滤（where 是分组前过滤，且不能用聚合函数）。

**3. 什么是索引**：加速查询的数据结构（多为 B+树），用空间和写入性能换查询性能。

**4. 哪些情况索引失效**：对索引列用函数/运算、隐式类型转换、`like '%xx'` 前导模糊、`or` 连接非索引列、不满足最左前缀（联合索引）。

**5. 什么是事务**：一组要么全成功要么全失败的操作单元。

**6. ACID**：原子性（不可分割）、一致性（前后数据合法）、隔离性（并发互不干扰）、持久性（提交后永久生效）。

**7. 脏读 / 不可重复读 / 幻读**：脏读=读到别的事务未提交的数据；不可重复读=同一行两次读值不同（被别的事务改了）；幻读=同一查询两次返回行数不同（被别的事务增删了）。

**8. 如何用 explain 分析 SQL**：看 `type`（访问类型，全表 ALL 最差）、`key`（是否用上索引）、`rows`（扫描行数）、`Extra`（是否 Using filesort / temporary），据此优化。

**9. 什么是数据库锁**：并发控制机制，有行锁/表锁、共享锁（读）/排他锁（写）、乐观锁/悲观锁。

**10. 如何定位慢 SQL**：开慢查询日志找出慢语句 → `explain` 分析执行计划 → 看是否缺索引/全表扫描/大量排序 → 优化索引或改写 SQL。

## Linux / Docker / Git

### Linux

**1. 实时查看日志**：`tail -f app.log`。

**2. 查找含关键字的日志**：`grep "ERROR" app.log`，配合 `-n`（行号）`-C 5`（上下文）。

**3. 查看端口占用**：`netstat -tunlp | grep 8080` 或 `lsof -i:8080`、`ss -tunlp`。

**4. 查看 CPU 和内存**：`top` / `htop`（实时），`free -h`（内存），`vmstat`。

### Docker

**5. 进入容器**：`docker exec -it <容器> /bin/bash`。

**6. 查看容器日志**：`docker logs -f <容器>`。

**7. 镜像 vs 容器**：镜像是只读模板（类比"类"）；容器是镜像的运行实例（类比"对象"），可启停删。

### Git

**8. merge vs rebase**：merge 保留分支历史、产生合并节点；rebase 把提交"嫁接"到目标分支顶端，历史线性更干净，但 **不要对已推送的公共分支 rebase**。

**9. reset vs revert**：reset 回退指针、改写历史（适合本地）；revert 生成一个反向提交来抵消，**不改历史**（适合已推送的公共分支）。

**10. 如何解决冲突**：`git pull` 后冲突文件出现 `<<<<<<< / ======= / >>>>>>>` 标记，手动保留正确内容删掉标记，`git add` 标记已解决，再 `git commit`（merge）或 `git rebase --continue`（rebase）。
