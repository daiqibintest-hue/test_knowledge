# 十五、AI 工程化工具（Q41-Q43）

---

## Q41：AI 工程化工具应该怎么细分

工程化工具不要只背工具名，要按 **AI 应用生命周期** 拆：从开发调试、数据入库、编排执行、评测验证，到上线监控、安全治理和成本优化。

**1. 开发调试工具**：Prompt Playground、Prompt 版本管理、Chain / Agent 调试器、Tool Call 调试、本地 Mock 和沙箱环境。核心价值是让 prompt、工具调用、上下文拼装可复现、可对比。

**2. 数据与知识库工具**：文档解析、数据清洗、Chunk 切分、Embedding 生成、向量库管理、索引构建、知识库版本管理。RAG 效果很大程度由这层决定。

**3. 编排与工作流工具**：Chain 编排、Agent Workflow、状态机 / 图编排、工具注册与调用、人工审批节点、异步任务。生产环境里常用 Workflow 管稳定流程，用 Agent 处理开放问题。

**4. 评测工具**：Prompt 回归测试、RAG 检索评测、答案质量评测、Agent 任务成功率评测、安全 / 越狱测试、幻觉检测、自动化测试集管理。

**5. 可观测性工具**：请求日志、Trace 链路追踪、Token 消耗、延迟、成本、用户反馈、线上质量看板。AI 应用排查问题必须能还原 query、召回片段、prompt、模型输出和工具调用轨迹。

**6. 模型管理工具**：模型路由、多模型 fallback、模型版本管理、A/B 测试、灰度发布、本地模型部署。重点是按场景在质量、成本、延迟之间做权衡。

**7. 部署与服务化工具**：API 网关、鉴权、限流、缓存、队列、容器化、Kubernetes、Serverless。AI 服务不能只看模型，还要看并发、超时、重试和降级。

**8. 安全与治理工具**：敏感信息检测、内容安全审核、权限控制、数据脱敏、审计日志、合规管理。企业 RAG 和 Agent 必须防越权、防泄露、防误操作。

**9. 成本优化工具**：Token 压缩、Prompt 缓存、语义缓存、模型降级策略、批处理、成本报表。上线后成本经常是 AI 应用能否持续运行的关键。

面试里可以一句话总结：**LLMOps 是把大模型应用从 demo 变成可测试、可监控、可治理、可持续迭代的工程体系**。

## Q42：AI 工程化工具如何接入测试开发流程

可以按测试开发的主链路接入：**需求理解 -> 用例生成 -> 脚本生成 -> 执行调度 -> 质量评测 -> 报告归因 -> 线上监控**。

**1. 需求阶段**：接入 PRD / Jira / Wiki，利用 RAG 查历史需求、历史缺陷和测试规范，辅助生成测试点，但关键业务规则必须人工确认。

**2. 用例阶段**：用 LLM 生成用例草稿，评测工具检查正常 / 异常 / 边界覆盖率，和 Golden Set 或人工基线做差异分析。

**3. 脚本阶段**：Agent 或 Skill 生成 pytest / Playwright 脚本骨架，再通过 lint、单测、沙箱执行和人工 review 后入库。

**4. 执行阶段**：Workflow 负责编排固定步骤，Agent 只处理需要判断的分支；执行 pytest、JMeter、Playwright 等工具时记录工具调用参数、结果和耗时。

**5. 评测阶段**：RAG 看 context recall / precision，答案看 faithfulness / relevance，Agent 看任务成功率、轨迹正确率和工具调用准确率。

**6. 监控阶段**：线上记录 query、召回、prompt、输出、token、延迟、用户反馈、拒答率和人工接管率，形成质量看板。

**7. 治理阶段**：危险操作加审批，生产环境默认只读，敏感数据脱敏，所有工具调用进入审计日志。

核心原则：**AI 可以提效，但不能绕过测试开发原有的质量门禁**。

## Q43：LLMOps 和传统测试平台工程化有什么区别

两者都强调工程化，但 LLMOps 多了 **模型不确定性、上下文链路、知识库版本、工具调用风险和成本治理**。

**1. 传统测试平台更关注确定性执行**：用例、脚本、环境、任务调度、报告、质量门禁都是明确规则驱动，输入相同通常输出相同。

**2. LLMOps 更关注概率输出的稳定性**：同一个 prompt、同一份知识库、不同模型版本或 temperature 都可能影响结果，所以要做 prompt 版本、评测集和回归基线。

**3. RAG 需要管理知识链路**：不仅要看最终答案，还要看文档解析、chunk、embedding、检索、rerank、引用来源和知识库版本。

**4. Agent 需要管理执行轨迹**：不能只看最终结果，还要审计每一步调用了什么工具、传了什么参数、失败后如何重试、是否越权。

**5. AI 应用需要成本和延迟治理**：Token、模型路由、缓存、批处理、降级策略会直接影响上线成本和用户体验。

**6. 安全边界更复杂**：Prompt Injection、越权检索、工具误调用、敏感信息泄露都需要专项测试和运行时防护。

回答时可以落到实践：传统测试平台解决 **怎么稳定跑测试**，LLMOps 解决 **怎么稳定运行和持续优化 AI 应用**。
